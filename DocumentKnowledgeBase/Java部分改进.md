# Java部分改进

本文档用于说明 `DocBase 企业文档知识库系统` 在 Java 后端层面的改造思路，重点体现“从通用后台底座重构为企业知识库系统”的工程化能力。

## 1. 项目定位

本项目不是从零新建后台，而是在成熟通用后台底座之上完成企业化重构，目标是：

1. 将原有通用管理后台改造成企业文档知识库系统。
2. 保留用户、角色、菜单、权限、日志、配置等底座能力作为业务基础设施。
3. 围绕知识分类、文档管理、版本管理、权限控制、AI 检索问答逐步扩展核心业务能力。

## 2. 现有系统底座能力分析

当前系统底座已经具备以下可复用能力：

- 用户、角色、部门、岗位管理
- 菜单、按钮权限、动态路由
- 登录认证、JWT、验证码
- 操作日志、登录日志、在线用户监控
- 系统配置、通知公告
- 文件上传、缓存、Redis、数据权限

这些能力不需要推倒重来，而是作为 `DocBase 企业文档知识库系统` 的后台基础设施继续复用。

## 3. 改造重点

需要重点新增或强化的业务能力：

- 知识分类树
- 文档元数据管理
- 文档版本管理
- 文档访问控制 ACL
- 文档解析/导入任务
- AI 会话与审计日志

## 4. 结构改造思路

保留的模块：

- `docbase-common`
- `docbase-domain`
- `docbase-infrastructure`
- `docbase-admin`

裁剪或弱化的部分：

- 非企业知识库场景下无直接价值的模板功能
- 与当前项目无关的开放接口设计
- 模板型外链菜单和宣传文案

## 5. 面试表达建议

在面试中，可以把这部分改造成果总结为：

1. 基于通用后台底座完成企业文档知识库系统重构，而不是简单套模板。
2. 对原系统做了模块裁剪、命名重构、技术升级和业务领域扩展。
3. 在保留后台通用能力的前提下，新增知识库核心模型与权限体系。
4. 通过配置、菜单、权限、数据库、最小骨架联调，逐步完成系统落地。

## 6. 关键工程亮点

- Java 版本升级到 17
- Spring Boot 升级到 3.2.6
- Spring Security 6 适配
- `javax` 迁移到 `jakarta`
- 包名统一调整为 `com.docbase`
- 模块名称统一调整为 `docbase-*`
- 数据库命名统一调整为 `docbase_knowledge`
- 登录、注册、菜单、权限链路按企业后台场景重新整理
