## 三、数据权限（DataScopeEnum）

以下用 5 个不同角色的用户分别验证：

### 3.1 ALL（管理员）
- [ ] 查看所有文档（不限部门/创建人）
- [ ] 查看所有分类（不限部门）
- [ ] 查看所有入库任务

### 3.2 CUSTOM_DEFINE（dept_id_set="2,4"）
- [ ] 只能看到部门 2 和部门 4 的文档/分类
- [ ] 看不到部门 3 的文档/分类
- [ ] 可以看到自己创建的文档（即使在其他部门）

### 3.3 SINGLE_DEPT（dept_id=2）
- [ ] 只能看到部门 2 的文档/分类
- [ ] 看不到部门 4 的文档/分类（即不包含子部门）
- [ ] 可以看到自己创建的文档

### 3.4 DEPT_TREE（dept_id=2，深圳总公司）
- [ ] 可以看到部门 2（深圳总公司）的文档/分类
- [ ] 可以看到部门 4（研发部门，parent=2）的文档/分类
- [ ] 可以看到部门 6（测试部门，parent=2）的文档/分类
- [ ] 看不到部门 3（长沙分公司，parent=1）的文档/分类

### 3.5 ONLY_SELF
- [ ] 只能看到自己创建的文档
- [ ] 只能看到自己创建的入库任务
- [ ] 同部门其他人创建的文档不可见

## 四、文档 visibility 规则

### 4.1 全员可见（visibility=1）
- [ ] 已发布（status=3）后，跨部门用户可见
- [ ] 未发布（status≠3）时，非创建人不可见

### 4.2 本部门可见（visibility=2）
- [ ] 已发布后，同部门用户可见
- [ ] 已发布后，跨部门用户不可见
- [ ] DEPT_TREE 角色的用户可见子部门的"本部门可见"已发布文档
- [ ] 创建人永远可见（无论 status 和 visibility）

### 4.3 仅本人可见（visibility=3）
- [ ] 已发布后，只有创建人可见
- [ ] 同部门其他用户不可见
- [ ] 管理员可见（ALL 数据范围）

---

## 五、审核流程

### 5.1 权限
- [ ] 普通用户无法审核任何文档
- [ ] 同部门审核员可审核本部门"审核中"文档
- [ ] 跨部门审核员不可审核其他部门文档
- [ ] DEPT_TREE 审核员可审核子部门文档

### 5.2 状态流转
- [ ] 审核通过 → status=3（已发布）
- [ ] 审核驳回 → status=4（已驳回）
- [ ] 非"审核中"状态文档无法审核（抛异常）

### 5.3 通知
- [ ] 审核通过后，通知公告出现"您的文档「xxx」已被审批通过"
- [ ] 审核驳回后，通知公告出现"您的文档「xxx」已被审批驳回"
- [ ] 审核通知创建失败时，审核操作本身不受影响
