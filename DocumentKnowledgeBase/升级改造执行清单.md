# 升级改造执行清单

本文档用于说明 `DocBase 企业文档知识库系统` 的升级与重构执行路径。

## 1. 总体目标

把原有通用后台工程改造成一个适合简历展示、可面试讲解、可继续扩展的企业文档知识库系统。

## 2. 已完成方向

1. 工程命名从模板风格统一调整为 `DocBase`
2. Java / Spring Boot / Spring Security 版本升级
3. 包名、模块名、启动类命名统一收口
4. 菜单、权限、前端占位页面接入
5. 数据库增加知识库相关核心表设计
6. 注册能力恢复，登录页模板入口裁剪
7. 模板外链菜单与模板宣传文案清理

## 3. 后续执行顺序

1. 完善知识库后端最小骨架
2. 完善知识分类、文档、版本、导入任务接口
3. 完善前端页面与接口联调
4. 接入 AI 问答与审计能力
5. 清理无用模板功能和历史文件

## 4. 模块定位

- `docbase-common`
  通用工具、基础能力、异常、配置、公共组件

- `docbase-domain`
  业务应用服务、领域模型、命令对象、查询对象

- `docbase-infrastructure`
  数据访问、缓存、三方集成、底层技术实现

- `docbase-admin`
  后台 Web 层、登录认证、系统管理与知识库管理入口

## 5. 面试表达建议

可重点强调以下几点：

1. 不是简单换皮，而是围绕企业文档知识库场景做了系统性重构。
2. 重构覆盖了命名、模块、权限、菜单、数据库、注册登录、工程升级。
3. 在原有底座基础上完成了知识库领域建模和企业化裁剪。
